How to Configure Mireka

Mireka is using the Resin application server for configuration and startup. Therefore it is distibuted in a standard Java Web Application archive, although currently it has no web interface. In the unlikely event, that some additonal configuration of Resin is needed, other than what was already done during installation, you can find more information about Resin on its web site at http://www.caucho.com/products/resin/. In the following only configuration options related to Mireka itself are described.

Mireka configuration files are located in the ${mirekaHome}/conf/ directory.

mireka.xmlconfigures the SMTP service, the filter chain, and most filter parameters.
domains.xmllist of local domain names. If the domain part of a recipient is not in this list, then the recipient will be rejected.

This file is imported by mireka.xml.

users.xmlsimple user addresses, like john@example.com. If a recipient equals with a user adress supplied here then it will be accepted (assuming other conditions are also met).

This file is imported by mireka.xml.

addresses.xmlmore complex address specificatons, like "any recipient which includes bounce in its local part, and of which remote part is lists.example.com".

This file is imported by mireka.xml.

logback.xmlconfigures logging. For more information, see the documentation of Logback.

The import structure of configuration files should be considered as an example. It is possible to include all data in mireka.xml. It is also possible to divide the configuration into more files than what is supplied by default.
